Background and history

History

The development of the current automated decision support has started in 2016.

In July 2017 a first prototype was shown at the UNCITRAL 50th Session in Vienna. This version focused on dispute resolution.

Beginning 2018 new tools and standards which became freely available on the market made it possible to shift focus from an IT tool to a datamodel. This free-ed up resources to broaden the legal scope to in-house legal counsel support.

 Features

Fingerprint technology

The datamodel ships with a fingerprint for each combination of answers. This enables to program a webtool to 'jump over' questions for which the answers are already known.

This unique feature is expected to be a great time-saver, and will motivate decision makers to use the tool.

Interactive questionnaire

The datamodel is build in such a way that when a question will not lead to an answer it will not be asked.

This highly efficient method enables parties to make one large model, being ensured that for simple issues only a few answers will be asked.

 Technology

Web ontology

The logical decision model is generated in the programming langualge "Ontology Web Language". An open source language.

Decision logic

The model is checked with a reasoner that works with decision logic.

JSON-Schema

The datamodel is build following the webstandard 'JSON-Schema'. For many of the programming languages were websites are build in, there are modules that can make nice webforms based on this webstandard.

Windows / Mac programs - prototype

Shortly Legal LinQ will present a stand-alone windows program for running the datamodel
